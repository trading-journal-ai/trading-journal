Layout

Negative Space

Space is an active part of the design system. It tells the user what belongs together, what can be scanned, and where to slow down.

4-8 Micro gaps inside pills, metric lines, ticker rows, and compact controls.
12-16 Form rows, filter groups, button groups, chart labels, and stat label-to-value rhythm.
24-32 Header-to-content gaps, filter-to-content gaps, and chart title-to-plot spacing.
44-56 Major breaks between stats, P&L, charts, calendar sections, and report modules.
64+ Editorial journal breaks where the user is reading, reflecting, and writing.

Level-up rule

Keep density inside the thing being scanned, then give the whole thing room around it. Trades can be tight. Ticker rails can be tight. Stat rows can be compact. But the spaces between filters, sections, charts, and journal entries should be generous enough that the page feels intentional instead of packed.

This is especially important in a trading journal: cramped pages feel reactive; open pages feel reflective.

Journal Prose column around 760-860px. Keep ticker rails close to the note, not floating at the far edge.
Reports Add clear section breaks between stats, P&L, and chart grids. Give two-up charts horizontal gutters.
Calendar The grid can be structured, but avoid heavy fills. Let text color and hairlines do most of the work.
Dashboard Remove extra rules under metric groups. Pull labels closer to values; separate only between sections.
Trades Ledger rows can stay dense, but filters, pagination, and range context need more breathing room.

Audit

Current App Gaps

These are the high-confidence deltas between the target system and the current app code.

Fix
Tokens are close but not aligned. Add page glow/prose tokens and converge background, surface, border, muted, and accent values.
Fix
Hard-coded colors still appear. Replace hard-coded blue and foreground values with tokens so theme changes are reliable.
Fix
Typography roles are documented, not implemented. Create reusable utilities or components for eyebrow, page title, stat label, stat value, and figure roles.
Fix
Controls are duplicated page by page. Segmented controls and ghost buttons should be shared primitives to prevent width, radius, and state drift.
Fix
Reports stats need matrix scanning. Detailed report metrics should use ReportsStatsMatrix so labels and values stay comparable and diagnostic.
Fix
Journal sidebar nav is too small. Promote month/week navigation to the Sidebar nav role at 13px so it reads as navigation, not micro annotation.
Close
Journal is closest to the north star. Use Journal as the reference when bringing Dashboard, Calendar, Reports, and Trades into the same language.
Implementation

Fix Order

Build the system through shared foundations first, then use those primitives to clean up pages without a full rewrite.

Foundations Update tokens, add page glow/prose, replace hard-coded colors.
Type Normalize titles, eyebrows, stat labels, stat values, and table headers.
Space Set consistent page widths, section breaks, chart gutters, and journal prose/ticker spacing.
Primitives Create shared Money, Eyebrow, SegmentedControl, GhostButton, StatBlock, ReportsStatsMatrix, and OpenSection.
Pages Apply primitives to Reports, Dashboard, Calendar, Trades, then Journal polish and states.
